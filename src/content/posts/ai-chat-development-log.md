# 项目背景

作为后台管理系统的一部分，我开发了一个类 ChatGPT 的 AI 对话模块。目标是让用户能够与多个智能体进行流畅的对话交互。

## 已实现功能 ✅

### 会话管理
- 会话列表分页加载，支持无限滚动
- 选择智能体后发送首条消息自动创建会话
- 会话重命名、删除、归档
- AI 自动生成会话标题（异步队列处理）

### 消息功能
- SSE 流式输出，实时显示 AI 回复
- 停止生成，保存已生成的部分内容
- 消息复制、删除、重新生成
- 点赞/点踩反馈
- 图片附件上传（多模态支持）
- Markdown 渲染 + 代码高亮

### 智能体系统
- 多智能体管理，每个智能体可配置不同模型
- 智能体搜索过滤
- 根据智能体 modalities 控制输入能力

### 会话切换优化
- 流式输出中切换会话，自动取消当前生成
- 取消时已生成内容保存到数据库
- 切回会话时加载已保存内容

### UI/UX
- 响应式布局，移动端适配
- 智能体优先的交互设计（左侧智能体列表，右侧对话区）
- 历史会话抽屉式展示
- 中英文国际化

## 技术架构

```
前端 (Vue3 + TypeScript)
├── composables/
│   ├── useAgents.ts          # 智能体管理
│   ├── useConversations.ts   # 会话管理
│   ├── useMessages.ts        # 消息管理
│   └── useStreamChat.ts      # 流式聊天核心
└── components/
    ├── AgentList/            # 智能体列表
    ├── ConversationDrawer/   # 历史会话抽屉
    ├── MessageList/          # 消息列表
    └── MessageInput/         # 消息输入

后端 (PHP/Webman)
├── AiChatModule.php          # 对话业务逻辑
├── AiChatService.php         # AI 调用服务
├── AiAgentModule.php         # 智能体管理
└── queue/
    └── GenerateConversationTitle.php  # 异步标题生成
```

### 流式输出流程

```
用户发送消息
    ↓
前端 SSE 连接后端
    ↓
后端调用 AI API（流式）
    ↓
逐 chunk 推送给前端
    ↓
前端实时更新消息内容
    ↓
完成后保存到数据库
```

## 心得体会

1. **流式输出是 AI 应用的标配**，用户体验提升明显
2. **异步队列处理非关键任务**，接口响应更快
3. **Composables 架构让代码更清晰**，单文件不要超过 500 行
4. **智能体优先的设计更符合用户心智**，用户想的是"用哪个助手"而不是"打开哪个会话"
