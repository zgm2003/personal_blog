## 项目经历

### 智澜管理系统（全栈独立开发）

**项目周期**：2025.06 - 至今  
**技术栈**：Webman + Vue3 + TypeScript + MySQL + Redis  
**项目地址**：[Gitee](https://gitee.com/zgm2003) | [博客](https://blog.zgm2003.cn)

**项目简介**：  
一套企业级后台管理系统，包含用户权限管理、AI 智能对话、数据可视化等模块。采用前后端分离架构，后端基于 Webman 高性能框架，前端使用 Vue3 + TypeScript 构建。

**核心职责与成果**：

**1. 架构设计与规范制定**
- 设计并实现 CMVD 分层架构（Controller → Module → Validate → Dep），制定《项目开发规范白皮书》
- Controller 层仅负责路由转发，Module 层处理业务逻辑，Dep 层封装数据访问，实现关注点分离
- 封装 BaseDep 基类，提供 `getMap`、`getMapActive` 等批量查询方法，统一解决 N+1 查询问题

**2. AI 智能对话模块**
- 独立完成类 ChatGPT 的 AI 对话系统，支持多智能体管理、流式输出、会话管理
- 基于 SSE（Server-Sent Events）实现流式响应，首字响应时间 < 500ms
- 实现流式输出中断机制：用户可随时停止生成，已生成内容自动保存
- 设计"智能体优先"的交互模式，优化用户体验
- 使用 Redis Queue 异步处理会话标题生成，接口响应时间从 3s 优化至 200ms

**3. 前端工程化实践**
- 封装 `useTable` Hook，统一处理列表页的加载、分页、搜索、删除等逻辑，减少 60% 重复代码
- 封装 `Search`、`AppTable` 等通用组件，支持配置化开发
- 实现 Axios Token 无感刷新机制，请求队列 + 自动重试，用户无感知
- 封装 `streamPost` 函数处理 SSE 流式请求，支持中断和错误处理

**4. 性能优化**
- 制定 N+1 查询解决规范，使用批量预加载 + keyBy 优化，列表接口查询次数从 N+1 降至 2-3 次
- 设计 Redis 缓存策略，区分静态数据（永久缓存）和业务数据（TTL + 主动清除）
- 优化页面加载，AI 对话页面接口调用从 4 个减少至 1 个

**5. 其他模块**
- 用户权限管理：RBAC 权限模型，支持菜单权限、按钮权限、数据权限
- 系统配置：字典管理、参数配置、操作日志
- 国际化：中英文双语支持，前后端统一 i18n 方案
